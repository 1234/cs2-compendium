# Contributing to CS2 Compendium

First: thank you. This project only gets better through people who find something
missing, wrong, or incomplete — and do something about it.

---

## What belongs here

This is a depth-first resource. A contribution is welcome if it:

- Explains something CS2-relevant that isn't already covered
- Corrects something that's wrong or outdated
- Adds depth to an existing section
- Adds a source that strengthens or challenges an existing claim

A contribution is **not** suitable if it:

- Promotes a specific product, service, or skin site
- Is based on anecdote without any supporting evidence
- Duplicates something already well-covered
- Is opinion presented as fact without acknowledging it's debated

---

## How to contribute

### Fixing something

If something is incorrect, outdated, or misleading:

1. Open an **Issue** describing what's wrong, what the correct information is,
   and ideally a source that supports the correction.

### Adding content

1. Fork the repository
2. Create your content in the relevant folder
3. Follow the structure: one folder per topic, one `README.md` as the main document
4. Open a **Pull Request** with:
   - A short description of what you're adding
   - Why it belongs in the compendium
   - Sources where applicable

### Updating outdated sections

CS2 is updated regularly — settings change, maps change, the meta changes.
If a section becomes outdated after a patch:

1. Open an Issue tagged `outdated` describing what changed
2. Or directly open a PR with the corrected information and a note about what changed

---

## Writing style

- **Explain the why.** Don't just say "set X to Y" — explain what happens if you don't.
- **Be honest about disagreement.** If pros, coaches or the community disagree on something,
  say so. Don't pretend there's always one right answer.
- **Primary sources over summaries.** If you're citing a pro's approach, link the interview.
  If you're citing a guide, link it.
- **No jargon without explanation.** Someone arriving from zero should be able to follow.
- **No sponsored or affiliate content.** Recommend things because they're good, not because
  someone paid for the placement.

---

## Tone

Direct, honest, depth-first. Not dumbed down, not condescending.
If something is genuinely hard to understand, explain it better — don't simplify it away.

---

## Reporting issues

Found a section that's wrong, incomplete, or out of date?
Open an Issue with the label that fits:

| Label | Use for |
|---|---|
| `wrong` | Factually incorrect information |
| `outdated` | Information that was correct but is no longer |
| `missing` | Something important that should be covered |
| `unclear` | Something that exists but is confusing |
| `update-request` | New content you'd like to see but can't contribute yourself |
